# 🎵 音频编辑器

## How to Run

### 使用 Docker Compose（推荐）

```bash
# 启动服务
docker-compose up -d

# 访问应用
# 浏览器打开 http://localhost:8081
```

## Services

| 服务名称 | 端口 | 说明 |
|---------|------|------|
| audio-editor | 8081 | 音频编辑器前端服务 |

## 测试账号

本项目为纯前端应用，无需登录，直接访问即可使用。

## 题目内容

现在希望开发一款 单机的 用于 做音乐简单 剪切和 2个音频合并的工具。使用纯前端技术 html css js是否可以实现。

## ✨ 功能特性

### 🎼 音频剪切
- 上传音频文件（支持 MP3、WAV、OGG 等格式）
- 可视化波形显示
- 拖拽或手动输入选择剪切范围
- 实时预览选区播放
- 导出剪切后的音频片段

### 🎹 音频合并
- 同时上传两个音频文件
- **顺序拼接**：音频1播放完后接着播放音频2
- **混音叠加**：两个音频同时播放
- 预览合并效果
- 导出合并后的音频

### 🎨 界面特点
- 现代简洁的深色主题
- 专业级波形可视化
- 直观的交互体验
- 响应式设计，支持移动端

## 📖 使用说明

### 音频剪切操作流程

1. **上传音频**
   - 点击或拖拽音频文件到上传区域
   - 等待文件加载和波形渲染

2. **选择剪切范围**
   - 方式A：在波形图上按住鼠标左键拖动选择
   - 方式B：在"开始时间"和"结束时间"输入框中手动输入

3. **预览选区**
   - 点击"预览选区"按钮试听选中的片段
   - 确认剪切范围是否正确

4. **导出音频**
   - 选择导出格式（WAV 或 MP3）
   - 点击"导出剪切音频"按钮
   - 浏览器会自动下载文件

### 音频合并操作流程

1. **上传音频文件**
   - 分别上传"音频1"和"音频2"
   - 查看波形预览

2. **选择合并模式**
   - **顺序拼接**：适合制作音频合集
   - **混音叠加**：适合添加背景音乐

3. **预览合并效果**
   - 点击"预览合并效果"按钮
   - 系统会生成合并后的音频
   - 可以播放试听

4. **导出音频**
   - 选择导出格式（WAV 或 MP3）
   - 点击"导出合并音频"按钮
   - 下载合并后的文件

## 🛠️ 技术栈

- **HTML5** - 页面结构
- **CSS3** - 样式设计（深色主题）
- **JavaScript (ES6+)** - 业务逻辑
- **Web Audio API** - 音频处理和播放
- **Canvas API** - 波形可视化
- **lamejs** - MP3 编码

## 🎯 核心模块说明

### AudioEngine (audioEngine.js)
- 音频文件加载和解码
- 音频剪切处理
- 音频合并（拼接/混音）
- 采样率转换和重采样

### WaveformRenderer (waveform.js)
- Canvas 波形绘制
- 选区可视化
- 播放头动画

### AudioExporter (exporter.js)
- WAV 格式导出
- MP3 格式编码和导出
- 文件下载处理

### UIManager (ui.js)
- 用户交互管理
- 拖拽上传
- 自定义下拉选择框
- Toast 消息提示

### AudioEditorApp (app.js)
- 整合所有模块
- 状态管理
- 事件绑定
